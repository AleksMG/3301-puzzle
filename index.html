Выявленные проблемы в предыдущей реализации:

  1. Детерминированное шифрование вместо случайного выбора из 16 омофонов на байт
  2. Некорректное дешифрование из-за ошибок в преобразованиях
  3. Использование hex-представления вместо работы с числовыми значениями
  4. Отсутствие гарантии уникальности омофонов
  5. Неправильное преобразование между байтами и координатами 3D пространства
  6. Возможность потери данных при обратном преобразовании

Внесенные исправления:

  1. Реализован случайный выбор одного из 16 омофонов для каждого байта
  2. Обеспечена полная обратимость преобразований (омофон → байт)
  3. Работа напрямую с числовыми значениями байтов (0-255) и омофонов (0-4095)
  4. Гарантирована уникальность всех 4096 омофонов
  5. Исправлены алгоритмы преобразования координат
  6. Добавлена проверка целостности данных при шифровании/дешифровании

Математическая основа алгоритма

Алгоритм шифрования:

  1. Для каждого байта B (0-255) определить 16 омофонов в кубе:
    H = { (x, y, z) | x, y, z ∈ [0,15], f(B, x, y, z) = 0 }
  2. Используя ключ K, выбрать случайный омофон из H:
    index = (B + K[i mod len(K)] + random) mod 16
  3. Преобразовать выбранную точку (x,y,z) в значение омофона:
    O = x × 256 + y × 16 + z

Алгоритм дешифрования:

  1. Для каждого омофона O вычислить координаты точки:
    x = O ÷ 256, y = (O mod 256) ÷ 16, z = O mod 16
  2. Восстановить исходный байт:
    B = f⁻¹(x, y, z)
  3. Проверить соответствие ключу

Формула преобразования байта в координаты:

Для байта B и индекса i ∈ [0,15]:

x = (B + i) mod 16

y = (B × 3 + i × 7) mod 16

z = (B × 5 + i × 11) mod 16

Формула восстановления байта из координат:

Для координат (x, y, z):

B = (x - i + 16) mod 16, где i определяется из y и z

i = (y - B×3 + 16) mod 16 = (z - B×5 + 16) mod 16
